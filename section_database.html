B+ Tree

B+树（B+ Tree）是一种自平衡的树数据结构，常用于数据库和文件系统中来存储和检索数据。它具有以下特点： 1. 所有值都在叶子节点：B+树的所有实际值都存储在叶子节点，而内部节点只存储键值用于索引和路径导航。 2. 叶子节点链表：所有叶子节点通过指针链接在一起，形成一个有序的链表，使得范围查询和顺序访问更加高效。 3. 多路平衡树：B+树是多路平衡树，内部节点可以有多个子节点，具体数量由树的阶（degree）决定。一个内部节点可以有最多`m`个子节点（`m`为树的阶），其中`m`通常较大。 4. 所有节点的深度相同：B+树是高度平衡的树，所有叶子节点的深度相同。这意味着从根节点到任何叶子节点的路径长度相同，从而保证了数据访问的时间复杂度稳定。 5. 分支因子高：由于B+树的分支因子高，它的高度通常较低，因此查找、插入和删除操作都可以在对数时间内完成（O(log n)）。 6. 节点分裂和合并：插入和删除操作可能导致节点分裂或合并，以保持树的平衡。例如，当一个节点超过允许的最大子节点数时，它会分裂成两个节点；当一个节点的子节点数少于最小值时，它可能会与相邻节点合并。 7. 冗余存储：内部节点存储的是索引键，而不是实际数据，因此B+树的内部节点相对较小，可以容纳更多的子节点。 8. 顺序访问效率高：由于叶子节点通过指针链表连接，B+树对顺序扫描和范围查询特别友好，效率很高。 通过这些特点，B+树在数据库系统中广泛应用，特别是用于实现索引（如MySQL的InnoDB存储引擎使用B+树来实现其索引结构）。

B+树删除一个结点； B+树增加一个结点。

自增主键

在MySQL中，自增主键（AUTO_INCREMENT）是一种常用的技术，用于生成唯一标识每一行的整数值。以下是关于使用自增主键的一些详细信息： 特点 1. 自动生成：每次插入新行时，MySQL会自动生成一个唯一的整数值，并将其分配给自增列。 2. 唯一性：自增主键保证了每一行的主键值是唯一的，从而确保每一行都有一个独特的标识符。 3. 顺序递增：默认情况下，自增主键的值是从1开始递增的整数，每次插入新行时递增1。这个默认值和递增步长可以通过配置改变。 创建自增主键 在创建表时，可以通过AUTO_INCREMENT关键字定义自增主键。例如： 


CREATE TABLE users (
    id INT AUTO_INCREMENT,
    name VARCHAR(100),
    email VARCHAR(100),
    PRIMARY KEY (id)
);

使用自增主键的优点 简化插入操作：不需要手动生成唯一标识符，简化了插入操作。 保证唯一性：自增主键自动确保每一行都有唯一标识符。 提高性能：整数型主键相对其他类型（如字符串）在索引和查询性能上更优。 使用自增主键的注意事项 避免主键重用：当删除行时，自增主键的值不会重用，除非显式重置自增计数器。 处理并发插入：在高并发插入的情况下，自增主键可能会遇到争用问题，需要考虑锁机制。 分布式环境：在分布式数据库环境中，自增主键可能需要特别处理，以避免不同节点之间的主键冲突。
